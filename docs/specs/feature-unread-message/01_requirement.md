# 需求规格 - 未读消息红点功能

## 功能概述

在私聊会话列表中显示未读消息数量的红色圆形徽章，用户查看消息后自动减少未读数量。

## 用户故事

作为用户，我希望在会话列表中能看到未读消息数量，这样我就知道哪些会话有新消息需要查看。

## 需求详述

### 功能范围 (In Scope)

1. **私聊会话未读消息**
   - 在左侧会话列表中显示未读消息数量
   - 未读数量以红色圆形徽章显示（红色圆圈 + 白色数字）
   - 数字为 1-99，超过 99 显示为 "99+"

2. **@ 提及消息未读**
   - 当收到 @ 提及消息时，增加未读计数
   - @ 提及消息视为重要消息，优先处理

3. **未读状态更新**
   - 用户切换到会话并查看消息后，清除该会话的未读计数
   - 阅读消息后，对应的未读数量减 1

### 功能范围 (Out Scope)

- 群组消息未读红点（暂不实现）
- 系统消息未读红点
- 消息撤回时的未读计数回退

## 验收标准 (AC)

### AC1: 显示未读消息红点

**给定** 用户有未读消息
**当** 进入聊天页面
**那么** 应在对应会话的会话项上显示红色圆形徽章

**验收细节**：
- 徽章位置：会话项右侧
- 徽章样式：红色背景 (#F56C6C)，白色文字
- 徽章大小：16px 高度，宽度根据数字自适应，最小 16px
- 徽章字体：12px，加粗
- 圆角：50% 或 8px

### AC2: 未读数量计算

**给定** 有 3 条未读消息
**当** 显示会话列表
**那么** 徽章显示数字 "3"

**验收细节**：
- 数量 1-99：显示具体数字
- 数量 >= 100：显示 "99+"

### AC3: 查看消息后清除未读

**给定** 会话有 2 条未读消息
**当** 用户切换到该会话
**那么** 未读数量应变为 0，徽章消失

**验收细节**：
- 切换会话即视为已读
- 徽章立即更新（无需等待）

### AC4: 收到新消息增加未读

**给定** 当前在其他会话
**当** 收到新消息
**那么** 对应会话的未读数量 +1，徽章更新

**验收细节**：
- 非当前会话收到消息，未读计数 +1
- 当前会话收到消息，未读计数不变
- 徽章实时更新

### AC5: 未读状态持久化

**给定** 页面刷新
**当** 重新进入聊天页面
**那么** 未读消息数量应保持刷新前的状态

**验收细节**：
- 使用 localStorage 存储未读状态
- 格式：`unread:convId:count`
- 读取时验证会话是否仍然存在

## 边界条件

| 条件 | 预期行为 |
|------|----------|
| 未读数量为 0 | 不显示徽章 |
| 未读数量为负数 | 强制设为 0 |
| 会话已被删除 | 清除对应的未读记录 |
| 离线状态 | 未读计数正常增加，刷新后保持 |

## 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| localStorage 不可用 | 在内存中维护未读状态（刷新后丢失） |
| 消息去重失败 | 重复消息不计入未读数 |
| 未读数据损坏 | 重建未读计数 |

## 性能要求

- 徽章更新延迟 < 100ms
- 未读状态读写 < 10ms

## 数据结构

### 未读计数存储

```typescript
interface UnreadCount {
  [conversationId: string]: number
}
```

### localStorage 格式

```
key: chat_unread_counts
value: {
  "private_abc123": 3,
  "group_def456": 0
}
```
