# 群组功能需求文档

## 1. 需求概述

实现完整的群组管理功能，支持创建群组、成员管理、群组消息等功能。

### 1.1 需求背景

当前系统支持私聊功能（chatType=2），但缺少群组管理功能。用户需要能够创建群组、邀请成员、发送群组消息。

### 1.2 现有架构分析

- **数据层**: Beanie ODM + MongoDB
- **架构模式**: 三层架构（Router → Service → Repository）
- **已有模型**:
  - `ChatLog`: 支持群聊（chatType=1）和私聊（chatType=2）
  - `User`: 用户模型
- **已有 DTO**: `WebSocketMessage.GroupInfo` 已定义群组信息结构
- **缺少**: `Group` 模型来持久化群组信息

## 3. 功能详细说明

### 3.1 创建群组

**业务规则**:
- 群组名称长度：1-50 字符
- 初始成员必须包含创建者
- 初始成员数量：2-100 人
- 创建者自动成为群主

### 3.2 邀请成员

**业务规则**:
- 只有群主可以邀请成员
- 被邀请用户必须存在
- 群成员上限：100 人
- 邀请成功后发送系统消息

### 3.3 移除成员

**业务规则**:
- 只有群主可以移除成员
- 不能移除自己（如需退出请使用"退出群组"）
- 移除后发送系统消息

### 3.4 退出群组

**业务规则**:
- 任何成员都可以退出
- 群主不能退出（如需退出请先转让群主或解散群组）

### 3.5 修改群组信息

**业务规则**:
- 只有群主可以修改群组信息
- 可修改：群名称、群头像 URL

### 3.6 解散群组

**业务规则**:
- 只有群主可以解散群组
- 解散后群组和相关消息保留但标记为已解散

### 3.7 群组消息

**业务规则**:
- 复用现有 `ChatLog` 模型（chatType=1）
- conversationId 使用 groupId
- 群成员可以发送和接收消息

## 4. 数据模型设计

### 4.1 Group 模型

```python
class Group(Document):
    # 基本信息
    name: str                    # 群组名称
    avatar: Optional[str]        # 群头像URL
    ownerId: str                 # 群主ID
    memberIds: List[str]         # 成员ID列表

    # 状态
    status: int                  # 状态(1-正常, 2-已解散)

    # 时间戳
    createAt: int                # 创建时间戳
    updateAt: int                # 更新时间戳
```

## 5. 接口清单

### 5.1 HTTP API

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | /v1/group/create | 创建群组 |
| GET | /v1/group/list | 获取群组列表 |
| GET | /v1/group/{id} | 获取群组详情 |
| POST | /v1/group/{id}/invite | 邀请成员 |
| POST | /v1/group/{id}/remove | 移除成员 |
| POST | /v1/group/{id}/exit | 退出群组 |
| PUT | /v1/group/{id} | 修改群组信息 |
| DELETE | /v1/group/{id} | 解散群组 |

### 5.2 WebSocket 消息

| 类型 | 描述 |
|------|------|
| group_create | 群组创建通知 |
| group_dismiss | 群组解散通知 |
| group_invite | 成员邀请通知 |
| group_remove | 成员移除通知 |
| group_exit | 成员退出通知 |

## 6. 验收标准 (AC)

### AC-1: 创建群组
- [ ] 可以成功创建群组
- [ ] 创建者自动成为群主
- [ ] 初始成员列表包含创建者

### AC-2: 获取群组列表
- [ ] 返回当前用户参与的所有群组
- [ ] 不包含已解散的群组

### AC-3: 邀请成员
- [ ] 群主可以邀请新成员
- [ ] 非群主不能邀请
- [ ] 成员上限校验生效

### AC-4: 移除成员
- [ ] 群主可以移除成员
- [ ] 非群主不能移除
- [ ] 被移除成员不再能接收群消息

### AC-5: 解散群组
- [ ] 群主可以解散群组
- [ ] 解散后群组状态更新
- [ ] 所有成员收到解散通知

### AC-6: 群组消息
- [ ] 群成员可以发送消息
- [ ] 所有成员都能收到群消息
- [ ] conversationId 正确使用 groupId

---

## 7. 边界条件和异常处理

| 场景 | 处理方式 |
|------|----------|
| 群组不存在 | 返回 404 |
| 用户无权限 | 返回 403 |
| 成员已存在 | 返回 400 错误 |
| 群组已解散 | 返回 400 错误 |
| 成员数量超限 | 返回 400 错误 |
| 群主退出 | 返回 400 错误 |

---

## 8. 性能要求

- 群组列表查询：< 500ms
- 创建群组：< 300ms
- 群组消息发送：< 100ms

---

## 9. 安全要求

- 验证当前用户是否为群组成员
- 验证操作权限（群主/成员）
- 输入参数校验（群名称长度、成员数量）
