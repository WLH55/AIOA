# 测试策略文档 - CRUD 业务接口实现

## 1. 测试概述

### 1.1 测试范围

本文档定义 6 个业务模块的测试策略，覆盖主流程、边界条件、异常情况。

### 1.2 测试类型

| 类型 | 覆盖率目标 | 说明 |
|------|-----------|------|
| 单元测试 | ≥ 80% | Service 层核心逻辑 |
| 集成测试 | ≥ 60% | API 端点到数据库 |
| 边界测试 | 100% | 边界条件和异常处理 |

## 8. 测试执行计划

### 8.1 单元测试优先级

| 优先级 | 模块 | 原因 |
|-------|------|------|
| P0 | User | 认证是所有接口的前置条件 |
| P0 | Department | 审批流程依赖部门结构 |
| P1 | Approval | 业务逻辑最复杂 |
| P1 | Todo | 核心业务功能 |
| P2 | ChatLog | 相对独立简单 |

### 8.2 集成测试优先级

| 优先级 | 场景 | 说明 |
|-------|------|------|
| P0 | 登录 → 创建待办 → 完成待办 | 核心用户流程 |
| P0 | 登录 → 创建审批 → 处理审批 | 核心审批流程 |
| P1 | 创建部门 → 添加用户 → 删除用户 | 部门管理流程 |

## 10. 自动化测试命令

```bash

pytest tests/ -v

# 运行指定模块测试

pytest tests/ --cov=app --cov-report=html
```
