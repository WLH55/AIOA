# 私聊消息转发功能 - 需求文档

## 1. 需求背景

当前 WebSocket 聊天模块的 `_handle_chat` 函数只返回模拟响应，无法正确处理私聊消息转发。私聊消息（chatType=2）应该转发给接收者，群聊消息（chatType=1）应该广播给群成员。

## 2. 需求描述

### 2.1 功能需求

| ID | 需求 | 优先级 |
|----|------|--------|
| REQ-1 | 区分私聊和群聊消息类型 | P0 |
| REQ-2 | 私聊消息转发给接收者 | P0 |
| REQ-3 | 群聊消息广播给群成员 | P0 |
| REQ-4 | 消息保存到数据库 | P0 |
| REQ-5 | 返回发送确认给发送者 | P1 |
| REQ-6 | 私聊会话ID按规则生成 | P1 |

### 2.2 验收标准 (AC)

**AC-1: 私聊消息转发**
- [ ] 发送私聊消息（chatType=2），接收者在线时能实时收到
- [ ] 接收者收到的消息包含：sendId, recvId, content, contentType, time
- [ ] 消息保存到 ChatLog 表

**AC-2: 群聊消息广播**
- [ ] 发送群聊消息（chatType=1），群内所有成员（除发送者）都能收到
- [ ] 消息保存到 ChatLog 表

**AC-3: 会话ID生成**
- [ ] 私聊会话ID格式：`private_{min(userId1, userId2)}_{max(userId1, userId2)}`
- [ ] 群聊会话ID = groupId

**AC-4: 边界处理**
- [ ] 接收者不在线时，消息仍保存到数据库
- [ ] 空消息内容不处理
- [ ] 非法chatType返回错误

## 3. 非功能需求

| 类型 | 要求 |
|-----|------|
| 性能 | 消息转发延迟 < 100ms |
| 可靠性 | 消息必须持久化，即使WebSocket失败 |
| 可扩展性 | 支持未来添加消息类型（如文件、语音） |

## 4. 边界条件

| 场景 | 处理方式 |
|-----|---------|
| 接收者不在线 | 保存消息，返回提示"对方不在线" |
| 发送者=接收者 | 返回错误"不能给自己发消息" |
| 消息为空 | 返回错误"消息内容不能为空" |
| 群聊不存在 | 返回错误"群聊不存在" |
| 非法chatType | 返回错误"无效的消息类型" |

## 5. 参考代码

| 文件 | 说明 |

| `app/routers/ws.py` | WebSocket 路由，需修改 `_handle_chat` |

| `app/models/chat_log.py` | 消息模型 |
