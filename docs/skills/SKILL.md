# Team Skills & Rules

AIWorkHelper 团队规则库 - 记录技术偏好、业务铁律、测试规范和错误记录。

---

## 技术偏好

### 编程语言
- **Python 版本**: >= 3.11
- **代码风格**: PEP 8 (snake_case 命名)
- **类型提示**: 所有函数必须包含类型注解
- **文档字符串**: Google 风格 docstring

### 框架和库
- **Web 框架**: FastAPI (异步优先)
- **ORM/ODM**: Beanie (MongoDB)
- **AI 框架**: LangChain + LangGraph
- **测试**: pytest + pytest-asyncio
- **代码格式化**: black
- **Linter**: ruff

### 命名约定

| 类型 | 约定 | 示例 |
|-----|------|------|
| 文件名 | snake_case | `auth_service.py` |
| 类名 | PascalCase | `UserService` |
| 函数/方法 | snake_case | `get_user()` |
| 常量 | UPPER_SNAKE_CASE | `MAX_RETRIES` |
| 私有成员 | 前缀下划线 | `_internal_method` |

## 业务铁律

### 数据处理
- **日期时间**: 统一使用 UTC 时区存储，datetime 类型
- **货币**: 使用 Decimal 类型，避免浮点精度问题
- **ID 字符串**: MongoDB 使用 ObjectId，对外暴露为 string

### 安全规范
- **密码**: 必须使用 bcrypt 哈希（work factor=12），不存储明文
- **JWT**: 使用 HS256 算法，密钥从环境变量读取
- **敏感数据**: 密码、Token 不在日志中输出
- **输入校验**: 所有用户输入必须通过 Pydantic 校验
- **SQL/NoSQL 注入**: 使用 Beanie ODM，避免原生查询

### API 设计
- **RESTful**: 遵循 REST 设计原则
- **HTTP 状态码**: 正确使用状态码（200/201/400/401/403/404/422/500）
- **错误响应**: 统一错误格式 `{detail, code, timestamp}`
- **版本管理**: URL 路径包含版本号 `/api/v1/`

### 分层边界

```
┌─────────────────────────────────────────────────────────────────┐
│  路由层                      │
│  ✅ 允许: 接收请求、参数校验、调用 Service                       │
│  ❌ 禁止: 业务逻辑、直接访问数据库                               │
├─────────────────────────────────────────────────────────────────┤
│  业务层                      │
│  ✅ 允许: 业务逻辑、调用 Repository、事务管理                    │
│  ❌ 禁止: HTTP 相关、直接访问外部 API                           │
├─────────────────────────────────────────────────────────────────┤
│  数据层             │
│  ✅ 允许: 数据库 CRUD、缓存操作                                │
│  ❌ 禁止: 业务逻辑、返回领域模型以外对象                         │
└─────────────────────────────────────────────────────────────────┘
```

## 测试规范

### 单元测试
- **测试框架**: pytest
- **覆盖目标**: Service 层 >= 90%，总体 >= 80%
- **Mock**: 使用 pytest-mock 进行依赖 Mock
- **命名**: `test_{function}_{scenario}`

```python
# 示例
def test_login_with_valid_credentials_returns_token(mock_repo):
    pass
```

### 集成测试
- **测试客户端**: httpx.TestClient
- **数据库**: 使用测试用 MongoDB 或 mongomock
- **清理**: 每个测试后清理数据，确保隔离
- **Fixture**: 使用 conftest.py 定义共享 fixtures

### 测试金字塔

```
        /\
       /E2E\        少量端到端测试
      /------\
     /  集成  \      中等数量集成测试
    /----------\
   /   单元测试  \   大量单元测试
  /--------------\
```

## 示例

### 2026-03-12 规则 #1: 文档驱动
- **规则**: 所有新功能必须先有 Spec 文档（SDD 工作流）
- **原因**: 防止上下文腐烂和维护断层

### 2026-03-15 规则 #2: 异步优先
- **规则**: 所有 I/O 操作必须使用异步（async/await）
- **原因**: FastAPI 是异步框架，阻塞操作会影响性能
- **示例**:
  ```python
  # 正确
  user = await user_repository.find_by_id(user_id)

  # 错误
  user = user_repository.find_by_id(user_id)  # 缺少 await
  ```

### 2026-03-15 规则 #3: 密码永不返回
- **规则**: API 响应中永远不包含 password_hash
- **原因**: 安全风险，即使哈希也不应泄露
- **示例**: 使用 ResponseDTO 排除密码字段

### 2026-03-15 规则 #4: 环境变量优先
- **规则**: 敏感配置必须从环境变量读取，不硬编码
- **原因**: 防止密钥泄露到代码仓库
- **示例**: SECRET_KEY、MONGODB_URI 等
